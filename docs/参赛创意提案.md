# ModelCred（模型可信度指数）参赛创意提案

> 适用赛事：TRAE AI 创造力大赛  
> 建议赛道：学习工作 / 造个新解法（或 社会服务 / 造种新体验）

## 1. 一句话介绍

ModelCred 是一个公开、社区共治的「大模型信誉分」站点：任何人都可以给模型打分并标注“幻觉 / 引用造假”等标签，让模型的可靠性被持续记录、可追踪、可讨论。

## 2. 问题与痛点

- 幻觉与引用造假常常“无成本”：模型出错后，用户几乎没有任何公开、可持续的反馈入口，最多只能私下吐槽。
- 单次评测越来越难解释真实使用体验：很多模型在 bench 上看起来不错，但换到真实工作流，尤其是不同 agent 脚手架组合下，表现差异非常大。
- 用户侧缺少第三方评价体系：大家都在重复踩坑，却没有一个能长期积累、可追踪、贴近真实使用场景的公共评测入口。

## 3. 解决方案概述

### 3.1 ModelCred 站点

- 厂商榜 + 模型榜：按 Provider / Cloud 分组展示，分数按时间衰减加权（半衰期 138 天），更贴近“近期可靠性”。
- 模型详情页：信誉分、30 天趋势、评分分布、相关事故、吐槽列表。
- 评分 API：`POST /api/ratings`，允许外部工具/脚本提交评分与标签。

### 3.2 配套 Skill（model-credibility-rater）

让用户在日常使用 AI 的过程中无需打开网站即可快速评分，把“发现幻觉”变成一次轻量但可沉淀的动作，也给用户一个明确的情绪出口和公开反馈入口。

隐私原则：

- 不读取任何对话内容，只提交用户主动确认的模型、星级、标签与可选评价。
- 默认不填写评价，用户自行决定是否公开吐槽。

## 4. 作品亮点（创新点）

- 从“评测”走向“信誉”：把模型可靠性当作一种长期累积的公共信号，而不是一次性跑分结果。
- 明确的责任提示：通过 Skill 把“本次输出会被评分/被记录”的信号带入真实使用场景。用户知道自己有反馈手段，模型也知道结果会被记录。
- 用户视角的真实 EVL：不仅看模型本身，还关注“模型 + agent 脚手架”的组合效果，例如不同版本模型在 `Claude Code`、`Kimi Code` 等环境中的真实体验差异。
- 兼顾功能性与传播性：前者帮助用户提升使用效果，后者通过“给模型打分”这件事的趣味性与情绪价值扩大社区影响。

## 5. Demo 演示方式（初赛可用）

### 5.1 本地运行

1. `cp .env.example .env`
2. `npm install`
3. `npx prisma migrate dev`
4. `npm run seed`
5. `npm run dev` 打开 `http://localhost:3000`

### 5.2 演示脚本（建议）

1. 打开首页：展示厂商榜、模型榜与“重大事故/最新吐槽”。
2. 打开某个模型详情页：展示信誉分、趋势、评分分布。
3. 通过接口提交一次“幻觉”扣分（或使用 Skill）：刷新页面即可看到分数与吐槽变化。

## 7. 后续计划（复赛/决赛方向）

- 引入更强的“证据锚定”：让评分与事故线索可附带可核查的引用/截图/链接。
- 加入社交媒体公开讨论汇集：抓取 Reddit、X 等平台关于模型的公开发言，补充 bench 之外的真实世界反馈。
- 数据持久化与上线：迁移到 PostgreSQL（如 Neon）以支持 serverless 部署的真实写入。
- 引入反刷/信誉机制：限制恶意评分，保护社区共治质量。
- 增加“事故线索提交/验证流程”：把事故从“展示”变成“协作核验”。
- 如果社区形成足够规模，进一步和模型厂商合作，提供更贴近真实使用体验的训练反馈与 EVL 数据。
