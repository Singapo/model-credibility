## 1. 创意名称 + 创意介绍

**创意名称：** ModelCred（模型可信度指数）

**想解决什么问题：** 现在大家用大模型时，经常会碰到幻觉、引用造假、答非所问，但用户几乎没有任何“处罚”或反馈它的渠道，最多只能私下吐槽。与此同时，传统 bench 对真实使用体验的解释力越来越有限，很多模型在真实工作流中的表现，和跑测试时完全不是一回事。

**为什么会想到做这个：** 我一方面观察到，模型如果明确知道自己会被记录、被评分，往往会更谨慎，这件事本身是有研究依据的；另一方面，作为用户，碰到模型幻觉时那种“明明错了却拿它没办法”的感觉很强，所以我想做一个公开的、可持续积累的评分工具，既给用户一个表达和宣泄的出口，也把这些真实体验慢慢沉淀成长期可用的评测数据。

**大概是什么产品：** 这是一个公开的大模型评分网站，配套一个可随手打分的 Skill；后续还会加入对 Reddit、X 等平台公开讨论的汇集，并结合不同 agent 脚手架与模型组合的实际效果，形成一个第三方、贴近真实使用体验的评测社区。

## 2. 目标用户及痛点

**面向哪些用户：** 主要面向重度使用 AI 的学习者、开发者、产品经理、研究者，以及依赖模型完成写作、编程、调研和工作任务的用户。

**在什么场景下使用：** 用户在写方案、查资料、写代码、跑 agent 工作流、比较不同模型和脚手架效果时，都可以用它记录真实体验。比如对我自己来说，就能明显感受到 `Kimi 2.6 + Kimi Code` 和 `Kimi 2.7 + Claude Code` 在不同工作流里的表现差异，这种差异往往是传统评测里看不出来的。

**当前痛点：** 如果没有这个产品，用户只能把体验零散地发在群聊、社交媒体或脑子里，无法沉淀为长期可追踪的数据；也很难系统比较“什么模型配什么 agent 更好用”。结果就是，大家都在重复踩坑，却没有形成真正属于用户侧的第三方评测。

## 3. 价值与意义

**效率提升：** 它能帮助用户更快判断“什么模型在什么任务里更靠谱”，尤其是不同模型与不同 agent 脚手架组合时的真实表现，从而减少反复试错的时间成本。

**商业价值 / 长期价值：** 如果这个社区未来积累了足够多的真实评分、实际体验和评测数据，它不仅可以成为用户侧的第三方评价入口，也有机会和模型厂商合作，为他们提供更贴近真实世界的训练反馈和 EVL 评测数据。

**社会价值：** 它让普通用户第一次拥有了一个可以公开记录模型错误、表达不满、并形成公共压力的工具。它的传播方式也比较特别：一方面靠功能性吸引用户持续使用，另一方面又通过“给模型打分”这件事本身的趣味性和情绪价值扩大影响。
